## 0. 실행 규칙 (작업 세션 필독)

1. **모든 디자인·마크업·카피 작업은 `web-designer` 서브에이전트로 위임** (`.claude/agents/web-designer.md`, 프로젝트 CLAUDE.md 규칙). 메인 세션이 `src/site.html`을 직접 수정하지 말 것.
2. **편집 소스는 `src/site.html`뿐**. `index.html`/`dist/`는 빌드 산출물 — 직접 수정 금지. 예외: `build.js`(빌드 스크립트 자체), `robots.txt`/`sitemap.xml`/`assets/`(정적 파일 추가)는 메인 세션이 직접 다뤄도 된다.
3. **작업 후 검증 루틴** (모든 코드 작업 공통 완료 조건):
   - `node build.js` 성공
   - `NODE_PATH=/Users/yongmin/node_modules node render.js` → `pageerrors 0` + 섹션별 스크린샷 육안 확인
   - 모바일 뷰포트 확인: 부록 A의 `render-mobile.js` 사용 (390×844)
4. **커밋/푸시는 사용자 확인 후** 메인 세션이 처리. `CNAME` 파일 항상 유지.
5. **사실 정보 규칙 (중요)**:
   - 비용·정원·인력·차량·운영시간·지정번호 등 **사실 수치는 절대 지어내지 말 것**. §0-1 체크리스트에서 확인된 값만 반영.
   - 미확인 값은 `{{플레이스홀더}}`로 두되, **플레이스홀더가 남은 상태로 배포(push) 금지**.
   - 용어 주의: **"주야간보호" 사용 금지**(야간 서비스 미제공 시 표방 위험 — 검증에서 교정된 사항). "주간보호(데이케어)" 사용.
   - "물리치료"는 물리치료사 배치가 확인된 경우에만 표기(면허 업무). 미배치면 "기능회복훈련(신체활동 프로그램)"으로.
   - 배상책임보험은 의무가입이 아니므로 실제 가입 확인 후에만 표기.

## 부록 A — 모바일 렌더 검증 스크립트 (render-mobile.js)

저장소에는 데스크톱용 `render.js`만 있다. 모바일 검증이 필요하면 아래를 저장소 루트(또는 임시 폴더)에 생성해 사용:

```js
#!/usr/bin/env node
/* render-mobile.js — index.html 을 모바일 뷰포트(390x844)로 섹션별 스크린샷 */
const { chromium } = require('playwright');
const path = require('path');
(async () => {
  const file = process.argv[2] || 'index.html';
  const url = /^https?:/.test(file) ? file : 'file://' + path.resolve(file);
  const browser = await chromium.launch({
    executablePath: '/Applications/Google Chrome.app/Contents/MacOS/Google Chrome',
    args: ['--use-gl=angle', '--use-angle=swiftshader', '--ignore-gpu-blocklist']
  });
  const page = await browser.newPage({
    viewport: { width: 390, height: 844 }, deviceScaleFactor: 2, isMobile: true, hasTouch: true,
    userAgent: 'Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.0 Mobile/15E148 Safari/604.1'
  });
  const errs = [];
  page.on('pageerror', e => errs.push(String(e.message || e)));
  await page.goto(url, { waitUntil: 'load' });
  await page.waitForTimeout(1500);
  const ids = await page.evaluate(() => Array.from(document.querySelectorAll('.scene[id]')).map(s => s.id));
  for (const id of ids) {
    await page.evaluate(x => document.getElementById(x).scrollIntoView({ block: 'start' }), id);
    await page.waitForTimeout(2000);
    await page.screenshot({ path: 'mshot-' + id + '.png' });
  }
  await page.evaluate(() => window.scrollTo(0, document.body.scrollHeight));
  await page.waitForTimeout(1500);
  await page.screenshot({ path: 'mshot-footer.png' });
  console.log('scenes', ids.length, '| pageerrors', errs.length);
  if (errs.length) console.log(errs.slice(0, 8).join('\n'));
  await browser.close();
})().catch(e => { console.log('FATAL', e.message); process.exit(1); });
```

실행: `NODE_PATH=/Users/yongmin/node_modules node render-mobile.js [index.html | https://gracedaycare.co.kr]`
**주의**: `file://` 로컬 렌더에서는 WebGL 텍스처가 cross-origin 제한으로 로드되지 않아 pageerror 5건 + 배경 폴백이 나온다(라이브에서는 정상). 배경 전환까지 보려면 라이브 URL 또는 로컬 HTTP 서버(`python3 -m http.server`)로 렌더할 것.
